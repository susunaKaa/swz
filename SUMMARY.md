### 2. 遇到的问题及解决方案

#### 问题1：Git 安装与环境配置
**问题描述：**
在 Windows 系统上安装和配置 Git 环境时遇到了路径问题，Git 可执行文件路径未正确添加到系统环境变量中。

**解决方案：**
- 使用 winget 包管理器安装 Git
- 手动指定 Git 可执行文件的完整路径
- 重启终端以加载新的环境变量

#### 问题2：GitHub Actions 自动部署
**问题描述：**
GitHub Actions 配置文件监听的是 `main` 分支，但初始推送时使用了 `master` 分支，导致自动化部署未触发。

**解决方案：**
- 将本地分支重命名为 `main`
- 使用强制推送更新远程分支
- 确认 GitHub Actions 监听正确的分支

#### 问题3：Docker 容器开发环境
**问题描述：**
需要在本地避免安装 Ruby 和 Jekyll，直接使用容器进行开发。

**解决方案：**
- 创建 Docker Compose 配置文件
- 使用官方 Jekyll 镜像
- 配置目录挂载实现热重载

## 二、静态网站开发的规范要求

### 1. 内容合规要求

#### （1）原创内容原则
- 所有文章和项目描述必须为本人原创
- 引用他人内容必须注明来源
- 不得抄袭、剽窃他人成果

#### （2）法律法规遵守
- 不得发布违法、违规内容
- 尊重知识产权和版权
- 避免侵犯他人隐私权

#### （3）网络安全责任
- 不泄露个人敏感信息
- 不发布虚假信息
- 维护网络空间清朗

### 2. 引用规范要求

#### （1）技术文档引用
- 引用 Jekyll、Ruby 等官方文档时标注来源
- 使用他人代码示例时注明出处
- 遵循相应的开源许可证

#### （2）图片资源引用
- 使用免费或授权的图片资源
- 标注图片来源和作者
- 尊重图片版权

#### （3）参考资源记录
```
参考资源：
1. Jekyll 官方文档：https://jekyllrb.com/
2. GitHub Pages 官方文档：https://docs.github.com/en/pages
3. Docker 官方文档：https://docs.docker.com/
4. Sass 官方文档：https://sass-lang.com/
```

### 3. 协作分工要求（如有团队）
